# Backlog placeholder

`backlog/` is optional in this blueprint.

## Use it when

- the team plans work as `EPIC-*` and `TASK-*`
- the repository wants scriptable work item lookup
- the PR and evidence flow should be keyed by backlog identifiers

## Suggested shape

- `backlog/EPIC-01/README.md`
- `backlog/EPIC-01/TASK-0101.md`
- `backlog/EPIC-01/TASK-0102.md`

## Task-first teams

If the repository does not use a structured backlog, keep this directory as a placeholder and use `--title` plus optional `--source-path` with the same scripts.
