# Runtime skills

As skills abaixo orquestram o fluxo operacional da blueprint `backend-platform-coder-agnostic`.

- `adr-driven-design`: Use when the user needs architecture decisions or tradeoffs captured before implementation.
- `api-contract-design`: Use when the user needs interface or contract design without locking a specific stack.
- `data-structure-review`: Use when the user needs data shape, invariants, or state transitions reviewed.
- `integration-cut-planning`: Use when the user needs sequencing or integration boundaries defined.
- `failure-mode-and-rollout-review`: Use when the user wants failure modes, degraded behavior, or readiness reviewed.
- `release-evidence-pack`: Use when the user needs a release or readiness evidence pack before PR publication.
- `github-pr-ops`: Use when the user wants to publish or maintain the GitHub PR for this backend blueprint.
