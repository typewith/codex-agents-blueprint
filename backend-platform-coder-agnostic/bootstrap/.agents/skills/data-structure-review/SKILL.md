---
name: data-structure-review
description: Use when the user asks for a data-structure or invariant review.
---

Follow this workflow for data-structure review:

1. Map the important entities, states, or messages.
2. Review invalid states, ownership ambiguity, and state transition clarity.
3. Record findings in evidence or decision notes.

- Treat data shape as behavior, not only as implementation detail.
- Keep invariants explicit.
