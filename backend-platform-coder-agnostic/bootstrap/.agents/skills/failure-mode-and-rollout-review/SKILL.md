---
name: failure-mode-and-rollout-review
description: Use when the user asks for rollout, rollback, or failure-mode review.
---

Follow this workflow for failure-mode and rollout review:

1. Enumerate realistic failure scenarios.
2. Define degraded behavior, rollback posture, and human checks.
3. Make open risks explicit in evidence and PR notes.

- Do not describe readiness in vague terms.
- Treat degraded mode and rollback as first-class outputs.
