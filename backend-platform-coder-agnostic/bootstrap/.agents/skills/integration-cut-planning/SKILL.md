---
name: integration-cut-planning
description: Use when the user asks how to cut or sequence a backend integration safely.
---

Follow this workflow for integration cut planning:

1. Define the boundary between internal and external surfaces.
2. Capture sequencing assumptions and fallback expectations.
3. Record risky coupling or rollout dependencies explicitly.

- Do not let integration sequencing stay implicit.
- Prefer smaller, reviewable cuts when possible.
