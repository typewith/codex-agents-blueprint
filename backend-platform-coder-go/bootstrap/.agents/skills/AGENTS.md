# Runtime skills

As skills abaixo orquestram o fluxo operacional da blueprint `backend-platform-coder-go`.

- `adr-driven-design`: Use when the user needs service architecture decisions captured before implementation.
- `api-contract-design`: Use when the user needs API or message contract design for Go services.
- `go-service-slicing`: Use when the user needs package boundaries or service slicing in Go.
- `storage-and-schema-review`: Use when the user wants storage or schema risk reviewed for a Go service.
- `worker-and-concurrency-patterns`: Use when the user needs worker, goroutine, or concurrency pattern help.
- `release-evidence-pack`: Use when the user needs a release-ready evidence bundle for a Go backend change.
- `github-pr-ops`: Use when the user wants to publish or maintain the GitHub PR for a Go service work item.
