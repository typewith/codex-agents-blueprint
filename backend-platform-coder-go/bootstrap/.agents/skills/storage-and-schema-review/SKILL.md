name: storage-and-schema-review
description: Use when the user asks for a storage, schema, or persistence review.
---

Follow this workflow for storage and schema review:

1. Review persistence assumptions, schema changes, and compatibility impact.
2. Capture any rollback or manual migration requirement.
3. Escalate if the rollout path is weak or undocumented.

- Treat storage behavior as operational surface, not hidden detail.
- Do not approve unclear rollback posture.
