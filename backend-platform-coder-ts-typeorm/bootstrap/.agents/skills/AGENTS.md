# Runtime skills

As skills abaixo orquestram o fluxo operacional da blueprint `backend-platform-coder-ts-typeorm`.

- `adr-driven-design`: Use when the user needs architecture decisions captured before or during backend work.
- `api-contract-design`: Use when the user needs endpoint, DTO, or event contract design.
- `typeorm-data-modeling`: Use when the user needs entity, repository, query, or migration work in TypeORM.
- `migration-safety-review`: Use when the user wants migration risk reviewed before publication.
- `background-jobs-and-integrations`: Use when the user needs queues, jobs, workers, or third-party integration changes.
- `release-evidence-pack`: Use when the user needs a backend evidence bundle before opening the PR.
- `github-pr-ops`: Use when the user wants to publish or maintain the GitHub PR for this backend blueprint.
