# Runtime skills

As skills abaixo orquestram o fluxo operacional da blueprint `design-system-coder`.

- `prototype-brief-to-mock`: Use when the user wants to turn a brief or prototype spec into a code-first mock.
- `code-first-mock-generation`: Use when the user wants to generate or refine code-first mocks quickly.
- `component-inventory-and-token-audit`: Use when the user needs a component or token audit before growing the mock.
- `prototype-parity-review`: Use when the user wants to compare the mock against the prototype contract.
- `visual-evidence-pack`: Use when the user needs screenshots, notes, and visual proof bundled for review.
- `github-pr-ops`: Use when the user wants to publish, review, or update the GitHub PR for the work item.
