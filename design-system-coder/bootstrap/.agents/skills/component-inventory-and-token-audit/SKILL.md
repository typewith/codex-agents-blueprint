description: Use when the user wants to map reusable pieces, tokens, and inconsistency hotspots.
---

Follow this workflow for inventory and token audit:

1. Inspect the current component surface and shared token assumptions.
2. Group components by role, state complexity, and likely reuse value.
3. Document missing tokens, duplicated patterns, and risky visual drift in `artifacts/review/`.

- Do not invent a full design system when the work item only needs a thin mock.
- Record audit findings in concrete terms, not vague style commentary.
