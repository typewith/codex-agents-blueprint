# Runtime skills

As skills abaixo orquestram o fluxo operacional da blueprint `mobile-app-coder`.

- `screen-brief-to-mock`: Use when the user wants to turn a screen brief into a mobile mock or screen flow.
- `expo-app-workflow`: Use when the user wants to implement or refine Expo/React Native screens and flows.
- `android-studio-ops`: Use when the user needs Android Studio, emulator, or Android-native guidance.
- `xcode-ops`: Use when the user needs Xcode, simulator, or iOS-native guidance.
- `screen-parity-review`: Use when the user wants to review screen parity against the prototype contract.
- `api-contract-sync`: Use when the user needs to align mobile state with backend contracts.
- `mobile-evidence-pack`: Use when the user needs a mobile evidence bundle before opening the PR.
- `github-pr-ops`: Use when the user wants to publish or maintain the GitHub PR for a mobile work item.
